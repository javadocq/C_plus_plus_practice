### Class

- 객체(object)가 생성되기 위한 틀
  -> 객체가 가져야 할 데이터와 기능을 정의
- 사용자 정의 "자료형"(user-defined data-type)
- 멤버 변수를 가진다(데이터)
  -> 속성, 필드, 클래스 변수 = 멤버 변수의 또다른 용어
- 멤버 함수를 가진다(함수, 동작)
  -> Method = 멤버 함수의 또다른 용어
- 데이터와 함수를 은닉 가능
- 인터페이스를 공개 가능

### Object

- 클래스로부터 생성된 객체
  -> 메모리에 올라간 객체는 인스턴스로 구분하여 명명하는 경우도 있다.
- 객체는 개별적으로 관리되며, 원하는 만큼 생성 가능하다.
- 객체를 통해 클래스에 정의된 멤버 함수를 호출 가능하다.

- 개념적인 예
  -> "철수"는 학생이라는 클래스의 객체
  -> "영희"도 학생이라는 클래스의 객체
  -> 철수와 영희는 각각의 학과, 학번, 키, 몸무게, 나이 등의 멤버 변수(데이터)를 가지고 있다.

### 클래스와 객체

- 클래스는 자료형(ex, int)처럼 사용함
- 클래스는 새로운 데이터 타입을 만드는 것이다.

```
int highScore;
int lowScore;

Player Kim;
Player Lee;
```

### 클래스 멤버의 접근

- 멤버 변수
- 멤버 함수
- 어떠한 멤버 변수 / 함수는 접근이 불가능 할 수 있다(정보 은닉 - private)
- 멤버 변수 / 함수에 접근하기 위해서는 객체가 필요하다.
  -> (Static 멤버의 경우는 예외)

- 클래스 객체인 경우
  - "." 연산자 사용 (멤버 접근 연산자)

- 객체의 포인터인 경우
  - 역참조 후 점 연산자 사용

  ```
  Account *kim_account = new Account();

  (*kim_account).balance;
  (*kim_account).Deposit(1000.00);
  ```

  - 화살표 연산자 (member of pointer 연산자) 사용

  ```
  Account *kim_account = new Account();

  kim_account->balance;
  kim_account->Deposit(1000.00);
  ```
