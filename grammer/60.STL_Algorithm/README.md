### STL의 알고리즘

- 반복자에 의해 지정된 요소 집합에 대해 수행되는 알고리즘 (find, max, for_each,accumulate 등)
- 매우 다양한 알고리즘이 구현되어 있다.

- 알고리즘을 사용하기 위해 함수를 인자로써 제공해야 하는 경우가 존재
- 함수를 인자로써 제공하는 방법
  -> Functor
  -> Function Pointer
  -> Lambda expression(C++11)

### 알고리즘과 반복자

- #include <algorithm>
- 컨테이너에 따라서 적용할 수 있는 알고리즘이 다르다.
- 모든 알고리즘은 반복자를 인자로 필요로 한다.

### 반복자의 유효성

- 반복자가 유효하지 않을 수 있다.
- 예를 들어, iterator가 가리키고 있는 요소가 삭제된다면?
  -> iterator가 유효하지 않아진다.

### 알고리즘 예시 1. find

- find 알고리즘은 컨테이너 내 어떤 요소가 첫 번째로 나타나는 지점을 찾아준다.
- 해당 지점을 가리키는 반복자를 반환하거나, end()를 반환한다.
  -> end()를 반환하는 경우, 해당 요소가 컨테이너 내에 없다는 뜻
- find 알고리즘의 사용을 위해서는 요소들을 비교할 수 있어야 한다.
- operator== 이 사용되므로, 클래스의 경우 오버로딩이 필요하다.
  -> 오버로딩이 되어있어야만 예제처럼 사용 가능하다.

### 알고리즘 예시 2. for_each

- for_each 알고리즘은 컨테이너 내 각 요소를 인자로 함수를 호출

- 컨테이너의 각 요소를 제곱하는 경우의 예제

- Functor의 사용
  - 함수(처럼 사용 가능한) 객체
    -> () 연산자의 오버로딩

- function pointer의 사용
  - 함수의 주소값을 인자로 전달
  - square()와 square의 차이?
    -> 함수의 호출 / 함수의 주소값

- lambda expression의 사용
  - 익명 함수, 한 번 쓰고 버리겠다라는 뜻
