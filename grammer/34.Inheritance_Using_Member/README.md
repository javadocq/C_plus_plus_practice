### 기본 클래스의 멤버 함수 사용

- 유도 클래스는 기본 클래스의 멤버 함수를 직접 호출 가능
	-> Public / protected인 멤버 함수의 경우에 한함
- 유도 클래스는 기본 클래스의 멤버 함수를 오버라이드 또는 재정의 가능
- 다형성의 기초가 되는 이론

### 정적 바인딩

- 이전 그림대로라면, 유도 클래스와 기본 클래스에 같은 이름과 인자를 갖는 함수가 2개 존재
- 어떤 기준으로 호출할 함수를 결정할까? -> 타입을 기준으로 호출! (정적 바인딩)

Player p는 Player 클래스의 객체이므로, p.Talk()는 Player 클래스의 Talk() 함수를 호출

### 동적 바인딩

- 정적 바인딩은, 컴파일 시 어떤 함수가 호출될지를 결정하는 방식
- C++의 기본은 정적 바인딩

- 동적 바인딩은, 실행 시 어떤 함수가 호출될지를 결정하는 방식 (다음에 설명)

### Is-A 관계

- 기본 클래스 객체를 사용하는 곳에서는 항상 유도 클래스 객체를 사용 가능
- 유도클래스에 기본 클래스의 멤버가 모두 포함되기 때문
- Player 객체는 두 가지 타입을 모두 가진 상태가 된다! 그래서 기본 클래스에서 유도 클래스 객체 사용 가능
