### 상속

- 기존 클래스를 기반으로 새로운 클래스를 정의하는 것을 상속이라고 한다.
- 상속을 통해 기존 클래스의 속성과 메서드를 재사용할 수 있다.
- 클래스들 간의 공통 속성에 집중하는 설계 방법
- 기존 클래스의 행동(함수)를 수정하여, 새로운 클래스의 행동을 새로 정의 가능
	-> 기존 클래스의 행동을 수정할 필요가 없다.

- Player 예시 설계
	- Player: x, y,speed, hp, xp, Talk(), Move()
	- Enemy: x, y, speed, hp, gold, Talk(), Move()
	- NonPlayer: x, y, dialog(), Talk()
	- 행동(함수)은 클래스마다 다를 수 있다.

	- 상속을 사용하여 Player, Enemy, NonPlayer 클래스를 정의할 때, 공통 속성인 x, y, Talk()를 Entity 클래스에 정의할 수 있다.
	- 이렇게 되면 Player 객체는 두 가지 타입을 가진 상태가 된다.
		- Player는 Entity의 자식 클래스이면서, Player 클래스의 객체이기도 하다. (IS-A)4

### Terminology

- 기본 클래스: 부모 클래스, 슈퍼 클래스, 베이스 클래스
	-  상속의 대상이 되는 클래스

- 유도 클래스: 자식 클래스, 서브 클래스, 파생 클래스
	- 기본 클래스로부터 생성되는 클래스
	- 데이터와 행동을 기본 클래스에서 상속받는다.
